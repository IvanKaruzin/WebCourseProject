# Описание REST API проекта «Онлайн-кинобиблиотека»

## Общая информация

REST API предоставляет доступ к каталогу фильмов, отзывам, избранному и аутентификации пользователей.

| Параметр              | Значение                                  |
| --------------------- | ----------------------------------------- |
| Базовый URL           | `http://127.0.0.1:8000/api/`              |
| Формат данных         | JSON                                      |
| Аутентификация        | Token (DRF `TokenAuthentication`)         |
| Документация (Swagger)| `GET /api/docs/`                          |
| OpenAPI-схема         | `GET /api/schema/`                        |

---

## Аутентификация

Большинство операций чтения (каталог, отзывы) доступны **без авторизации**. Запись (создание отзыва, работа с избранным) требует токена.

После регистрации или входа клиент получает токен. Его нужно передавать в заголовке каждого запроса, требующего авторизации:

```http
Authorization: Token <ваш_токен>
```

### Уровни доступа эндпоинтов

| Обозначение         | Значение                                          |
| ------------------- | ------------------------------------------------- |
| Публичный           | Доступен всем, токен не нужен                      |
| Только чтение без токена | GET доступен всем, изменение — только с токеном |
| Требует авторизации | Без токена возвращается `401 Unauthorized`        |

---

## Пагинация

Списочные эндпоинты возвращают данные постранично (по **20** записей). Управление через query-параметры `?page=` и стандартный формат ответа:

```json
{
  "count": 1000,
  "next": "http://127.0.0.1:8000/api/movies/?page=2",
  "previous": null,
  "results": [ ... ]
}
```

# Эндпоинты аутентификации

## Регистрация

```http
POST /api/auth/register/
```

Доступ: **публичный**.

### Тело запроса

| Поле     | Тип    | Обязательное | Ограничения        |
| -------- | ------ | ------------ | ------------------ |
| username | string | да           | уникальный         |
| email    | string | нет          | формат email       |
| password | string | да           | минимум 8 символов |

```json
{
  "username": "ivan",
  "email": "ivan@example.com",
  "password": "Passw0rd!23"
}
```

### Ответ `201 Created`

```json
{
  "user": {
    "id": 1,
    "username": "ivan",
    "email": "ivan@example.com",
    "date_joined": "2026-06-02T10:15:30Z"
  },
  "token": "9c1f3e8a2b7d4f..."
}
```

### Ошибки

* `400 Bad Request` — имя пользователя занято, пароль короче 8 символов или невалидный email.

---

## Вход

```http
POST /api/auth/login/
```

Доступ: **публичный**.

### Тело запроса

| Поле     | Тип    | Обязательное |
| -------- | ------ | ------------ |
| username | string | да           |
| password | string | да           |

```json
{
  "username": "ivan",
  "password": "Passw0rd!23"
}
```

### Ответ `200 OK`

```json
{
  "user": {
    "id": 1,
    "username": "ivan",
    "email": "ivan@example.com",
    "date_joined": "2026-06-02T10:15:30Z"
  },
  "token": "9c1f3e8a2b7d4f..."
}
```

### Ошибки

* `400 Bad Request` — неверный логин или пароль.

---

## Выход

```http
POST /api/auth/logout/
```

Доступ: **требует авторизации**. Удаляет текущий токен пользователя (его нужно будет получить заново при следующем входе).

### Ответ `204 No Content`

Тело ответа пустое.

---

## Текущий пользователь

```http
GET /api/auth/me/
```

Доступ: **требует авторизации**. Возвращает данные владельца токена.

### Ответ `200 OK`

```json
{
  "id": 1,
  "username": "ivan",
  "email": "ivan@example.com",
  "date_joined": "2026-06-02T10:15:30Z"
}
```

# Эндпоинты фильмов

## Список фильмов

```http
GET /api/movies/
```

Доступ: **публичный**. Возвращает постраничный список фильмов в кратком виде.

### Query-параметры

| Параметр  | Тип    | Описание                                                          | Пример                     |
| --------- | ------ | ---------------------------------------------------------------- | -------------------------- |
| page      | int    | Номер страницы                                                   | `?page=2`                  |
| genre     | string\|int | Фильтр по жанру: название (без учёта регистра) или его id    | `?genre=Action`, `?genre=3`|
| title     | string | Поиск по подстроке в названии (без учёта регистра)               | `?title=godfather`         |
| search    | string | Синоним `title`                                                  | `?search=batman`           |
| ordering  | string | Сортировка. Допустимо: `imdb_rating`, `-imdb_rating`, `year`, `-year`, `title`, `-title` | `?ordering=-imdb_rating` |

Параметры можно комбинировать: `GET /api/movies/?genre=Action&ordering=-imdb_rating&page=1`.

### Ответ `200 OK`

```json
{
  "count": 1000,
  "next": "http://127.0.0.1:8000/api/movies/?page=2",
  "previous": null,
  "results": [
    {
      "id": 3,
      "title": "The Dark Knight",
      "year": 2008,
      "imdb_rating": 9.0,
      "poster_url": "https://m.media-amazon.com/images/...",
      "genres": [
        { "id": 1, "name": "Action" },
        { "id": 5, "name": "Crime" },
        { "id": 7, "name": "Drama" }
      ]
    }
  ]
}
```

## Поиск фильмов

```http
GET /api/movies/search/?title=<подстрока>
```

Доступ: **публичный**. Возвращает постраничный список фильмов, в названии которых встречается подстрока (без учёта регистра). Формат записей — как в списке фильмов.

### Пример

```http
GET /api/movies/search/?title=godfather
```

```json
{
  "count": 3,
  "next": null,
  "previous": null,
  "results": [
    { "id": 2, "title": "The Godfather", "year": 1972, "imdb_rating": 9.2, "poster_url": "...", "genres": [...] }
  ]
}
```

---

## Отзывы конкретного фильма

```http
GET /api/movies/{id}/reviews/
```

Доступ: **публичный**. Возвращает постраничный список отзывов на указанный фильм.

### Ответ `200 OK`

```json
{
  "count": 1,
  "next": null,
  "previous": null,
  "results": [
    {
      "id": 7,
      "user": "ivan",
      "movie": 3,
      "rating": 9,
      "text": "Отличный фильм",
      "created_at": "2026-06-02T11:00:00Z"
    }
  ]
}
```

# Эндпоинты отзывов

## Список отзывов

```http
GET /api/reviews/
```

Доступ: **публичный**. Возвращает все отзывы постранично.

### Query-параметры

| Параметр | Тип | Описание                          | Пример        |
| -------- | --- | --------------------------------- | ------------- |
| movie    | int | Фильтр отзывов по id фильма       | `?movie=3`    |

---

## Добавление отзыва

```http
POST /api/reviews/
```

Доступ: **требует авторизации**. Автор отзыва определяется автоматически по токену.

### Тело запроса

| Поле   | Тип    | Обязательное | Ограничения      |
| ------ | ------ | ------------ | ---------------- |
| movie  | int    | да           | id существующего фильма |
| rating | int    | да           | от 1 до 10       |
| text   | string | нет          | текст отзыва     |

```json
{
  "movie": 3,
  "rating": 9,
  "text": "Отличный фильм"
}
```

### Ответ `201 Created`

```json
{
  "id": 7,
  "user": "ivan",
  "movie": 3,
  "rating": 9,
  "text": "Отличный фильм",
  "created_at": "2026-06-02T11:00:00Z"
}
```

### Ошибки

* `400 Bad Request` — оценка вне диапазона 1–10 или несуществующий фильм.
* `401 Unauthorized` — запрос без токена.

---

## Удаление отзыва

```http
DELETE /api/reviews/{id}/
```

Доступ: **требует авторизации**.

### Ответ `204 No Content`

# Эндпоинты избранного

Избранное всегда привязано к текущему пользователю: клиент видит и изменяет только свои записи.

## Список избранного

```http
GET /api/favorites/
```

Доступ: **требует авторизации**. Возвращает постраничный список избранных фильмов текущего пользователя. Поле `movie_detail` содержит краткую карточку фильма.

### Ответ `200 OK`

```json
{
  "count": 1,
  "next": null,
  "previous": null,
  "results": [
    {
      "id": 4,
      "movie": 3,
      "movie_detail": {
        "id": 3,
        "title": "The Dark Knight",
        "year": 2008,
        "imdb_rating": 9.0,
        "poster_url": "...",
        "genres": [ { "id": 1, "name": "Action" } ]
      },
      "added_at": "2026-06-02T11:05:00Z"
    }
  ]
}
```

---

## Добавление в избранное

```http
POST /api/favorites/
```

Доступ: **требует авторизации**. Пользователь определяется по токену.

### Тело запроса

| Поле  | Тип | Обязательное | Ограничения             |
| ----- | --- | ------------ | ----------------------- |
| movie | int | да           | id существующего фильма |

```json
{
  "movie": 3
}
```

### Ответ `201 Created`

```json
{
  "id": 4,
  "movie": 3,
  "movie_detail": { "id": 3, "title": "The Dark Knight", "...": "..." },
  "added_at": "2026-06-02T11:05:00Z"
}
```

### Ошибки

* `400 Bad Request` — фильм уже в избранном (нарушение `UNIQUE (user, movie)`) или несуществующий фильм.
* `401 Unauthorized` — запрос без токена.

---

## Удаление из избранного

```http
DELETE /api/favorites/{id}/
```

Доступ: **требует авторизации**. Удаляет запись избранного по её id (не по id фильма).

### Ответ `204 No Content`

# Коды ответов

| Код | Значение                                                    |
| --- | ----------------------------------------------------------- |
| 200 | Успешный запрос (GET, login)                                |
| 201 | Ресурс создан (register, создание отзыва/избранного)        |
| 204 | Успех без тела ответа (logout, удаление)                    |
| 400 | Ошибка валидации входных данных                             |
| 401 | Запрос требует авторизации, но токен отсутствует/неверен    |
| 404 | Ресурс не найден                                            |

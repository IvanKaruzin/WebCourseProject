# Схема базы данных проекта «CineVault»

## Общая информация

База данных предназначена для хранения информации о фильмах, пользователях, актёрах, режиссёрах, жанрах, отзывах и избранных фильмах пользователей.

В качестве СУБД используется SQLite. Взаимодействие с базой данных осуществляется через Django ORM.

### Обозначения ключей и ограничений

| Обозначение | Значение                                                        |
| ----------- | --------------------------------------------------------------- |
| PK          | Первичный ключ (Primary Key)                                    |
| FK          | Внешний ключ (Foreign Key)                                      |
| UNIQUE      | Уникальное значение                                             |
| NOT NULL    | Поле обязательно на уровне БД (значение не может быть NULL)      |
| NULL        | Поле может содержать значение NULL                              |
| AUTO        | Значение генерируется автоматически                             |

Все таблицы имеют суррогатный первичный ключ `id` (Integer, PK, AUTO), создаваемый Django автоматически.

> **Примечание о текстовых полях.** Строковые и текстовые поля, объявленные в Django с `blank=True`, но без `null=True`, на уровне БД остаются `NOT NULL` и хранят пустую строку `''` вместо `NULL`. В таблицах ниже такие поля помечены `NOT NULL` с пометкой «необяз. (пустая строка)»: они не обязательны для заполнения пользователем, но физически не принимают `NULL`.

# ER-диаграмма (логическая структура)

```text
                        ┌──────────┐
                        │   User   │
                        └────┬─────┘
                  ┌──────────┴──────────┐
                  ▼                     ▼
              ┌────────┐          ┌──────────┐
              │ Review │          │ Favorite │
              └───┬────┘          └────┬─────┘
                  │                    │
                  ▼                    ▼
                ┌──────────────────────────┐
                │           Movie           │
                └───┬───────────┬───────────┤
       directors    │   actors  │   genres  │
       (M2M)        ▼   (M2M)    ▼  (M2M)    ▼
            ┌──────────────┐         ┌──────────┐
            │    Person    │         │  Genre   │
            └──────────────┘         └──────────┘
```

* `User → Review`, `User → Favorite` — связь «один ко многим».
* `Movie → Review`, `Movie → Favorite` — связь «один ко многим».
* `Movie ↔ Person` (directors) — связь «многие ко многим».
* `Movie ↔ Person` (actors) — связь «многие ко многим».
* `Movie ↔ Genre` — связь «многие ко многим».

# Таблица User

Хранит информацию о зарегистрированных пользователях системы. Используется стандартная модель `django.contrib.auth.models.User`.

| Поле         | Тип       | Ключ   | Ограничения       | Описание                              |
| ------------ | --------- | ------ | ----------------- | ------------------------------------- |
| id           | Integer   | PK     | NOT NULL, AUTO    | Уникальный идентификатор пользователя |
| username     | String    | UNIQUE | NOT NULL          | Логин пользователя                    |
| email        | String    |        | NOT NULL, необяз. (пустая строка) | Адрес электронной почты               |
| password     | String    |        | NOT NULL          | Хэш пароля                            |
| date_joined  | DateTime  |        | NOT NULL          | Дата регистрации                      |

### Связи

* `1 → N` с таблицей **Review** (один пользователь — много отзывов).
* `1 → N` с таблицей **Favorite** (один пользователь — много избранных записей).

---

# Таблица Movie

Хранит основную информацию о фильмах.

| Поле        | Тип        | Ключ | Ограничения    | Описание                        |
| ----------- | ---------- | ---- | -------------- | ------------------------------- |
| id          | Integer    | PK   | NOT NULL, AUTO | Уникальный идентификатор фильма |
| title       | String     |      | NOT NULL       | Название фильма                 |
| year        | Integer    |      | NULL           | Год выпуска                     |
| certificate | String     |      | NOT NULL, необяз. (пустая строка) | Возрастной рейтинг              |
| runtime     | Integer    |      | NULL           | Продолжительность в минутах     |
| overview    | Text       |      | NOT NULL, необяз. (пустая строка) | Краткое описание сюжета         |
| imdb_rating | Float      |      | NULL           | Рейтинг IMDb                    |
| meta_score  | Integer    |      | NULL           | Оценка Metacritic               |
| votes       | Integer    |      | NULL           | Количество голосов              |
| gross       | BigInteger |      | NULL           | Кассовые сборы                  |
| poster_url  | String     |      | NOT NULL, необяз. (пустая строка) | Ссылка на постер фильма         |

### Ограничение уникальности

* `UNIQUE (title, year)` — составное ограничение: не допускается два фильма с одинаковыми названием и годом выпуска.

### Связи

* `M ↔ N` с таблицей **Person** через `Movie_directors` (режиссёры).
* `M ↔ N` с таблицей **Person** через `Movie_actors` (актёры).
* `M ↔ N` с таблицей **Genre** через `Movie_genres` (жанры).
* `1 → N` с таблицей **Review** (один фильм — много отзывов).
* `1 → N` с таблицей **Favorite** (один фильм — много избранных записей).

# Таблица Person

Хранит информацию о людях, участвующих в создании фильма. Используется как для режиссёров, так и для актёров.

| Поле | Тип     | Ключ   | Ограничения    | Описание                 |
| ---- | ------- | ------ | -------------- | ------------------------ |
| id   | Integer | PK     | NOT NULL, AUTO | Уникальный идентификатор |
| name | String  | UNIQUE | NOT NULL       | Имя и фамилия            |

### Связи

* `M ↔ N` с таблицей **Movie** через `Movie_directors` (как режиссёр).
* `M ↔ N` с таблицей **Movie** через `Movie_actors` (как актёр).

---

# Таблица Genre

Справочник жанров фильмов.

| Поле | Тип     | Ключ   | Ограничения    | Описание                       |
| ---- | ------- | ------ | -------------- | ------------------------------ |
| id   | Integer | PK     | NOT NULL, AUTO | Уникальный идентификатор жанра |
| name | String  | UNIQUE | NOT NULL       | Название жанра                 |

Примеры значений:

* Action
* Drama
* Adventure
* Thriller
* Comedy
* Fantasy

### Связи

* `M ↔ N` с таблицей **Movie** через `Movie_genres`.

---

# Таблица Review

Хранит отзывы пользователей на фильмы.

| Поле       | Тип      | Ключ | Ограничения    | Описание                       |
| ---------- | -------- | ---- | -------------- | ------------------------------ |
| id         | Integer  | PK   | NOT NULL, AUTO | Уникальный идентификатор       |
| user_id    | Integer  | FK   | NOT NULL       | Автор отзыва (→ User.id)       |
| movie_id   | Integer  | FK   | NOT NULL       | Фильм (→ Movie.id)            |
| rating     | Integer  |      | NOT NULL       | Оценка пользователя от 1 до 10 |
| text       | Text     |      | NOT NULL, необяз. (пустая строка) | Текст отзыва                   |
| created_at | DateTime |      | NOT NULL, AUTO | Дата создания                  |

### Связи

* `N → 1` с таблицей **User** (`user_id`, при удалении пользователя — CASCADE).
* `N → 1` с таблицей **Movie** (`movie_id`, при удалении фильма — CASCADE).

---

# Таблица Favorite

Хранит список избранных фильмов пользователей.

| Поле     | Тип      | Ключ | Ограничения    | Описание                 |
| -------- | -------- | ---- | -------------- | ------------------------ |
| id       | Integer  | PK   | NOT NULL, AUTO | Уникальный идентификатор |
| user_id  | Integer  | FK   | NOT NULL       | Пользователь (→ User.id) |
| movie_id | Integer  | FK   | NOT NULL       | Фильм (→ Movie.id)      |
| added_at | DateTime |      | NOT NULL, AUTO | Дата добавления          |

### Ограничение уникальности

* `UNIQUE (user_id, movie_id)` — один и тот же фильм нельзя добавить в избранное одного пользователя дважды.

### Связи

* `N → 1` с таблицей **User** (`user_id`, при удалении пользователя — CASCADE).
* `N → 1` с таблицей **Movie** (`movie_id`, при удалении фильма — CASCADE).

Таблица реализует связь «многие ко многим» между пользователями и фильмами с дополнительным атрибутом `added_at`.

---

# Связь Movie ↔ Person (Режиссёры)

Тип связи: Many-to-Many. Реализуется промежуточной таблицей `Movie_directors`, создаваемой Django автоматически.

| Поле      | Тип     | Ключ   | Ограничения    | Описание           |
| --------- | ------- | ------ | -------------- | ------------------ |
| id        | Integer | PK     | NOT NULL, AUTO | Идентификатор связи |
| movie_id  | Integer | FK     | NOT NULL       | Фильм (→ Movie.id)  |
| person_id | Integer | FK     | NOT NULL       | Режиссёр (→ Person.id) |

* `UNIQUE (movie_id, person_id)` — пара «фильм–режиссёр» уникальна.

Один фильм может иметь несколько режиссёров, один человек может быть режиссёром нескольких фильмов.

---

# Связь Movie ↔ Person (Актёры)

Тип связи: Many-to-Many. Реализуется промежуточной таблицей `Movie_actors`, создаваемой Django автоматически.

| Поле      | Тип     | Ключ   | Ограничения    | Описание            |
| --------- | ------- | ------ | -------------- | ------------------- |
| id        | Integer | PK     | NOT NULL, AUTO | Идентификатор связи |
| movie_id  | Integer | FK     | NOT NULL       | Фильм (→ Movie.id)  |
| person_id | Integer | FK     | NOT NULL       | Актёр (→ Person.id) |

* `UNIQUE (movie_id, person_id)` — пара «фильм–актёр» уникальна.

Пример: фильм может иметь нескольких актёров (Elijah Wood, Ian McKellen, Viggo Mortensen), а актёр может участвовать во множестве фильмов.

---

# Связь Movie ↔ Genre

Тип связи: Many-to-Many. Реализуется промежуточной таблицей `Movie_genres`, создаваемой Django автоматически.

| Поле     | Тип     | Ключ   | Ограничения    | Описание           |
| -------- | ------- | ------ | -------------- | ------------------ |
| id       | Integer | PK     | NOT NULL, AUTO | Идентификатор связи |
| movie_id | Integer | FK     | NOT NULL       | Фильм (→ Movie.id) |
| genre_id | Integer | FK     | NOT NULL       | Жанр (→ Genre.id)  |

* `UNIQUE (movie_id, genre_id)` — пара «фильм–жанр» уникальна.

Пример: фильм «Властелин колец» может одновременно относиться к жанрам Adventure, Fantasy, Action.

# Использование датасета IMDb

Для первоначального наполнения базы данных используется датасет IMDb Top 1000 Movies and TV Shows.

Соответствие полей датасета структуре БД:

| Поле датасета | Таблица | Поле        | Примечание по обработке                      |
| ------------- | ------- | ----------- | -------------------------------------------- |
| Series_Title  | Movie   | title       |                                              |
| Released_Year | Movie   | year        | Приведение к Integer; битые значения чинятся |
| Certificate   | Movie   | certificate | Может быть пустым                            |
| Runtime       | Movie   | runtime     | Удаление суффикса « min», приведение к Integer |
| Overview      | Movie   | overview    |                                              |
| IMDB_Rating   | Movie   | imdb_rating | Приведение к Float                           |
| Meta_score    | Movie   | meta_score  | Может быть пустым                            |
| No_of_Votes   | Movie   | votes       | Приведение к Integer                         |
| Gross         | Movie   | gross       | Удаление разделителей-запятых; может быть пустым |
| Poster_Link   | Movie   | poster_url  |                                              |
| Director      | Person  | name        | Через связь Movie_directors                  |
| Genre         | Genre   | name        | Разбор по запятым; через связь Movie_genres  |
| Star1–Star4   | Person  | name        | Через связь Movie_actors                     |

При импорте автоматически создаются связи между фильмами, режиссёрами, актёрами и жанрами.
